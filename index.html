"Mahidol University for Real World Impact"

Vision 02

Regulatory Guillotine

ทำกฎระเบียบของมหาวิทยาลัยให้ทันสมัย มีประสิทธิภาพ
ใช้ 5Cs ของ Scott Jacobs
ใช้การประเมินผลสัมฤทธิ์ของกฎหมาย
(พ.ร.บ.หลักเกณฑ์การจัดทำร่างกฎหมายและการประเมินผลสัมฤทธิ์ของกฎหมาย 2562)

Input
Process (5Cs)
Output

กระบวนการกิโยติน

  1. Input: รับ Feedback + ผลประเมินสัมฤทธิ์ + สังเกตการณ์
  2. Process: ประมวลผลด้วย Algorithm 5Cs
  3. Output: กฎระเบียบที่ตอบโจทย์ ทันสมัย ใช้งานได้จริง

Cut (ยกเลิก)

หั่นหรือยกเลิกกฎหมาย กฎระเบียบ หรือขั้นตอนที่ซ้ำซ้อน ไม่จำเป็น ไม่คุ้มค่าความเสี่ยง หรือถูกยกเลิกการใช้งานไปแล้วออกจากระบบ

Combine (ควบรวม)

นำระเบียบที่มีความคล้ายคลึงกัน มารวมกันเพื่อลดขั้นตอนการติดต่อหลายหน่วยงาน สู่บริการ One Stop Service

Change (เปลี่ยนแปลง)

ปรับปรุงขั้นตอนต่างๆ ให้ง่ายขึ้น สะดวกขึ้น และสอดคล้องกับโลกยุคปัจจุบัน

Continue (คงไว้)

หลังทบทวนแล้ว หากพบว่าสิ่งที่มีอยู่นั้นดีและเหมาะสมอยู่แล้ว ให้ดำเนินการใช้ต่อไป

Create (สร้างใหม่)

ออกกฎระเบียบใหม่ๆ ให้รองรับบริบททางสังคมใหม่ นวัตกรรม หรือเทคโนโลยี

Vision 04

ลดขั้นตอนด้วย Lean-Agile

สร้างความแตกต่างจากระบบราชการเดิม
เน้นกระจายอำนาจ กำกับดูแลที่ผลลัพธ์
ใช้อำนาจในการกำกับดูแลและตรวจสอบ มากกว่าใช้อำนาจบังคับบัญชาขีดเส้นให้เดิน
กำหนดเพียงข้อห้ามที่จำเป็นตามระดับความเสี่ยง

Lean (กำจัดความสูญเปล่า)

ตัดทอนขั้นตอนที่ไม่จำเป็น งานซ้ำซ้อน งานล่าช้า เพื่อสร้างกระบวนการที่ไหลลื่น (Flow) และเพิ่มคุณค่าให้ผู้รับบริการสูงสุด

Agile (ยืดหยุ่น รวดเร็ว)

แบ่งงานชิ้นใหญ่ออกเป็นส่วนย่อยๆ แล้วทยอยทำทีละส่วนให้เสร็จสมบูรณ์ทีละชิ้น เมื่อนำมารวมกันจะได้เป็นชิ้นงานที่สมบูรณ์ (Incremental) เพื่อส่งมอบงานได้อย่างรวดเร็ว ปรับเปลี่ยนแผนได้ตามสถานการณ์ ให้ความสำคัญกับผลลัพธ์และคนมากกว่ากระบวนการ

การออกแบบระบบกฎหมาย (Governance Design)

ระบบราชการเดิม

"ขีดเส้นให้เดิน"

ต้องทำตามกฎระเบียบทุกขั้นตอนอย่างเคร่งครัด เดินออกนอกเส้นไม่ได้ ขาดความยืดหยุ่น ล่าช้า

แนวคิดใหม่

"สร้างถนนให้กว้าง"

กำหนดเพียงข้อห้ามที่จำเป็น
ผู้ปฏิบัติงานสามารถเลือกเลนเดินทาง/วิธีการได้อิสระ
(ทำอะไรก็ได้ แต่อย่าฝ่าฝืนข้อห้าม)
และเน้นผลลัพธ์
